CONNECTION SETUP

Your API key stays in this browser (localStorage) and is sent only to the provider you choose. Nothing passes through any third-party server.

Runs fully in your browser via WebGPU — no key, no server, your data never leaves your machine. Needs a recent Chrome or Edge. First run downloads the model (hundreds of MB to a few GB) and caches it; smaller models load faster but reason less sharply than the hosted providers above.

For your own machine only — never for a public repo: this app stores keys in your browser's localStorage automatically, which is fine for personal use but resets per-browser. If you want the key to travel with the files instead, two options:

1. config.js (recommended if you fork this repo) — create a config.js file next to this one (see config.example.js), add it to .gitignore, and it will auto-load on startup. Your real key never touches git.

2. Obfuscated single-file bake (private repos only) — generates a scrambled string you paste into index.html. This is obfuscation, not real security: anyone who opens this page's dev tools can recover the key in seconds. Never use this in a public repository.
